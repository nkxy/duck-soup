Field Restrictions

Team: 1 designer, 1 product manager, 3 devleopers, 1 QA

Role: Product Design

Duration: January - April 2020

Responsibilities: Interaction design, visual design, prototyping, usability testing, development handoff

The challenge

Veeva is a software company that focuses on life sciences and pharmaceutical applications. OpenData is Veeva's database that customers can purchase access to. It contains data about health care providers and organizations, that pharmaceutical companies might be interested in.

Currently, once a customer has been given access to OpenData they can see all of the data inside of the database. However, OpenData now wants the ability to make certain data fields premium, so that only specific customers who have bought those fields can access them.

OpenData Admins use Veeva Network, a data management tool, to configure OpenData. The goal is to create a feature that allows OpenData admins to set up and configure these premium fields inside Network for OpenData users.
